Thank you for the clarification. You're right — the screenshots I sent previously showed Activity API surfaces. Here are the Health API surfaces.

StrideIQ uses Health API data differently than Activity API data. Activity data (runs, splits, GPS) is displayed directly on activity pages. Health API data (sleep, HRV, resting heart rate, stress) is not primarily a display feature — it is studied over time alongside training data to discover where health metrics are meaningfully impacting athlete performance, either positively or negatively.

The Health API data feeds our intelligence and coaching systems. It surfaces to the athlete only when there is a genuine, data-backed insight — not as raw numbers on a dashboard. This means:

- Sleep duration and quality from the Garmin device are correlated with next-day running performance over weeks and months. When a pattern is confirmed (e.g., "your efficiency improves within 3 days when sleep is higher"), the athlete is told.

- HRV, resting heart rate, and stress scores are used to assess recovery readiness and injury risk. The coach references these metrics when an athlete asks about their training load or race preparation.

- Device-measured sleep is compared against athlete self-reported sleep to identify discrepancies and provide more accurate coaching context.

Where this appears in the product:

1. Home page morning briefing — references Garmin device-measured sleep and uses it in coaching context (screenshot 1: mobile view showing "6.1 hours on the Garmin last night" with device vs. self-reported sleep comparison)

2. Home page coaching insights — correlations discovered from Health API inputs drive personalized insights about training adaptation, sleep impact, and race readiness (screenshot 2: desktop view showing sleep-driven coaching insight and race readiness assessment)

3. AI Coach — when an athlete asks about recovery or race readiness, the coach references Garmin Health API metrics including overnight HRV, resting heart rate, stress score, device sleep duration, and sleep score (screenshot 3: coach session showing HRV 30ms, resting HR 55bpm, Garmin stress 25/100, device sleep 5.6h, sleep score 54/100 — all used to assess Tobacco Road marathon readiness)
