## What the Next Builder Should Know

### The canary is the active watch item
The founder's morning briefing is now being generated by `kimi-k2-turbo-preview`. First real test is tomorrow morning. If anything sounds generic, wrong in tone, or misses context — rollback immediately per runbook. After 3-5 days clean, the decision point is full rollout vs. dual-model.

### Pre-commit discipline that slipped this session
Before committing any change to a constant, **grep the test suite for that constant name first**. The `COACH_MAX_OUTPUT_TOKENS` test failure was avoidable. The pattern: `grep -r "CONSTANT_NAME" apps/api/tests/` before every commit that changes a constant value.

### After every push: launch a CI watcher
Do not close the loop until CI is green. Do not ask the founder to check CI. Use the `ci-watcher` subagent — it polls to completion and returns the result.

### The "use markdown" contradiction is fixed
The base `SYSTEM_INSTRUCTIONS` previously said "Format responses with clear structure (use markdown)" while the spec wanted conversation-style prose. That line is gone. All three prompts now consistently say "write like a coach talks, never use tables/headers/emojis."

### Coach tool-call loop is on Sonnet — do not change this without a spec
`_query_opus_with_tools` in `ai_coach.py` uses Anthropic's tool-use format. kimi-k2-turbo-preview uses OpenAI-compatible function calling. These are not interchangeable. Any migration of the coach tool loop requires a dedicated offline parity spec and explicit founder approval.
