## What this advisor got wrong — READ THIS FIRST

### Mistake 1: Manual share cards (Phase 2)

I recommended building shareable finding cards on the Personal Operating Manual page. The founder literally never visits that page. I scoped a feature on an unused surface. The cards themselves were also creatively bad — raw UTM URLs, jargon, dead space. The builder executed my spec faithfully; the spec was wrong.

**Cost:** Builder time + my advisory time. Cards have been disconnected (commit `0e9aec6`).

**Lesson:** Before scoping any share/growth feature, verify the target surface is one the founder and users actually visit daily. Ask first.

### Mistake 2: "Disable Runtoon"

After the founder expressed frustration with Runtoon's output quality (animals, generic captions), I told the builder to comment out `<RuntoonSharePrompt />` globally. The founder correctly called this out — they spent money building Runtoon and "just throw it away" is not a fix.

**Cost:** Builder time on the disable, then a revert. Trust damage.

**Lesson:** Runtoon's concept (personalized shareable post-run visual) is sound and is exactly the kind of PLG lever this product needs. The problem is output quality — the prompts, the visual concept (random animals), the captions. The fix is a creative reset, not suppression. The Runtoon disable has been reverted (commit `8ce1fb3`).

This is a good candidate for GPT 5.4 if it has visual design sense. The pipeline (Gemini image gen, Celery task, R2 storage, share modal) all works — it's the creative direction that's broken.

### 4. Coach intelligence gap — researched, not built
The builder traced the call chain:
- `interval_detector.detect_interval_structure` serves `GET /v1/activities/{id}/splits`
- Home briefing uses `_summarize_workout_structure` (different pipeline)
- `moment_narrator` enriches stream-derived moments but doesn't consume interval summary

The gap: structured interval detection and the coach/briefing narration use different data pipelines. The coach can miss an obvious interval session because it never receives the `interval_summary`. Fix requires a product decision about unifying detection or explicitly feeding interval data into coach context.

### 5. Subscriber acquisition — the real problem
The founder has ~16 registered users. The product has genuine intelligence (correlation engine, fingerprinting, weather normalization) but no traffic. The phased growth plan exists (`docs/GROWTH_PHASED_PLAN.md`). Phase 1 (SEO + telemetry) is deployed. Phases 2-4 need rethinking after the share card failure.

The founder's insight: they already manually screenshot the run shape, splits, and morning brief to share. The growth lever is making those surfaces look so good that organic screenshots drive "what app is that?" questions. Not dedicated share features on unused pages.
