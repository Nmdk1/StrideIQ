## Why This Exists

The $5 one-time unlock currently removes blur on screen. Runners need a portable artifact (print/save/share) to perceive real product value. PDF export is the artifact that turns "paywall removal" into "plan purchase."

---

## Objective

Ship v1 PDF export for training plans with existing entitlement rules.

**Endpoint:** `GET /v1/plans/{plan_id}/pdf`  
**Frontend entry point:** Download button on `/plans/[id]`  
**Entitlement gate:** reuse `can_access_plan_paces()` (no new tier logic)

---

## Scope (In)

### PDF Content Contract

**Page 1 — Plan Summary**
- StrideIQ branding
- Athlete name
- Plan name
- Race distance, goal date, total weeks, start/end dates
- Baseline RPI (if present)
- Pace reference card with all 5 zones in both min/mi and min/km
- Generation timestamp

**Pages 2+ — Weekly Plan Tables**
- Week header: `Week N — Phase`
- Table columns: Day | Workout Type | Title | Distance | Pace Target | Notes
- Rest days shown as explicit rows
- Phase transitions visually separated
- No completed/skipped status (this is the plan, not activity history)

**Footer on every page**
- `Generated by StrideIQ — strideiq.run`
- page numbering

### Access Rules

- Free: 403
- One-time purchaser: 200 for purchased plan
- Guided: 200 for active plans
- Premium: 200 for active plans

All access behavior must come from `can_access_plan_paces()`.

## Required Safeguards

1. **Filename safety**
   - sanitize `plan_name` in `content-disposition`
   - prevent quotes/slashes/control chars
2. **Ownership semantics**
   - return 404 for missing/non-owned plan
   - return 403 for owned-but-not-entitled plan
3. **No external fetches in PDF render**
   - template must not depend on remote assets
4. **Generation guardrails**
   - timeout protection and bounded in-memory generation
   - graceful error path if generation fails
5. **Unit display rule**
   - weekly tables use athlete preferred unit only
   - pace reference card shows both min/mi and min/km

---

## WeasyPrint Decision

Use WeasyPrint for v1.

Reason:
- HTML/CSS template is maintainable and faster to iterate
- better table/pagination control than manual coordinate drawing
- one-time image size increase is acceptable for this slice

If future infra pressure appears, split PDF generation into dedicated service later (not in this scope).
