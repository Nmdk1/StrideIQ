## How to Use

### For AI Coaching Engine
These documents provide the knowledge base for generating coaching recommendations. The AI should:
1. Reference these principles when making suggestions
2. Never prescribe, only show patterns
3. Adapt to individual athlete data
4. Use neutral terminology (no coach names)

### For Product Documentation
Can be adapted for:
- User-facing educational content
- Help documentation
- Blog posts
- Marketing material

### For Development
Provides the domain knowledge needed to:
- Classify workouts correctly
- Generate training plans
- Identify warning signs
- Calculate appropriate zones

---

## Governing Principle

> **"Suit plans to athletes, not athletes to plans."**

Every recommendation must be filtered through the athlete's individual response, not population averages.
